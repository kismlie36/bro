CORE DEFENSE PROTOCOLS

核心
防禦
機制

SentineIx #3108 採用多層行為分析引擎，實時監控訊息流量與使用者行為模式。系統不依賴關鍵字過濾，而是透過時間窗統計、內容重複度檢測、帳號屬性評估等綜合指標，判斷是否構成威脅。所有判斷邏輯均可配置，並支援自訂閾值，以適應不同社群規模與活躍度。

執行原則：最小權限運作、不儲存任何聊天內容、僅記錄必要事件 ID 與處置記錄。所有自動行動均可在伺服器 Audit Log 中完整追蹤。

⚡ 刷屏與洪水攻擊防護

監控短時間內訊息頻率與內容相似度。當單一使用者在指定時間窗內超過可配置的訊息上限，或連續發送高度重複內容時，系統將自動刪除超限訊息，並累計警告次數。警告達到預設閾值後，執行暫時隔離（timeout），並記錄事件至指定 log 頻道。隔離期間使用者無法發送訊息，有效阻斷洪水攻擊擴散。

🤖 惡意機器人識別與封鎖

自動掃描伺服器內所有機器人帳號，評估註冊時間、使用者名稱模式、頭像狀態、行為特徵。發現高風險特徵（如新註冊、無頭像、名稱包含已知攻擊關鍵詞、發送可疑連結或提及全體成員）時，立即執行封禁，並將事件同步至跨伺服器黑名單資料庫。所有封禁行動均附帶詳細原因記錄，可供管理員審核。

🔐 聯防機制與驗證鎖定

支援跨伺服器黑名單共享：任何一台 SentineIx 標記的惡意帳號，將自動同步至所有使用相同系統的伺服器。新成員加入時可強制執行按鈕驗證流程，未完成驗證者無法獲得完整權限。緊急情況下可啟動 Lockdown 模式，一鍵關閉全伺服器發言通道，防止攻擊進一步擴散。

不儲存任何聊天內容或私人訊息，僅保留必要帳號 ID 與事件元資料。所有資料傳輸採用端到端加密，系統設計符合最小資料收集原則。
